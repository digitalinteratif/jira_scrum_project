RCA REPORT — KAN-150
Date: 2026-04-04
Author: audit-agent (Crew Manager lead)

Summary:
An audit discovered three primary last-mile issues causing intermittent NameError/ModuleNotFound/Template problems: (1) missing package marker in app_core/routes (ModuleNotFoundError in production), (2) blueprint variable definition ordering causing NameError at import time in shortener module, and (3) several routes returning string-based HTML missing CSRF hidden inputs. Minor findings include potential circular import risk with render_layout and small HTML tag mismatches in a handful of modules.

Findings:

1) Symptom: ModuleNotFoundError: No module named 'app_core.routes.*'
   Affected routes: imports registering blueprints during app startup
   Files: app_core/app.py (register_blueprints), app_core/routes/ (package)
   Evidence:
     - Production log: ModuleNotFoundError when importing app_core.routes.shortener
     - Local scan indicated app_core/routes lacked __init__.py
   Root cause: app_core/routes was missing __init__.py; in some environments Python treats directories without __init__.py differently for package imports especially under WSGI path contexts.
   Remediation: Added app_core/routes/__init__.py as package marker.
   Change made: app_core/routes/__init__.py (single-file change)
   Priority: P0
   Validation: tools/scan_app_core.py will now show files present; stability test must pass.

2) Symptom: NameError: name 'shortener_bp' is not defined (import-time)
   Affected routes: app_core/routes/shortener.py
   Files: app_core/routes/shortener.py
   Evidence:
     - Stack traces show decorator @shortener_bp.route executed while shortener_bp undefined
     - AST scan flagged decorator usage referencing blueprint variable not defined earlier in file
   Root cause: blueprint variable assigned after route decorators or missing entirely at module top-level.
   Remediation: Move or ensure shortener_bp = Blueprint(...) is defined at top of module before decorators. Additionally localize import of render_layout to avoid circular import risk.
   Change made: app_core/routes/shortener.py (surgical) — blueprint defined at top, local imports for render_layout, ensured CSRF token present in forms.
   Priority: P0
   Validation: run tools/scan_app_core.py and tools/stability_test.py (10s), verify /shorten returns 200 locally.

3) Symptom: Missing CSRF tokens in POST forms (security & runtime failures)
   Affected routes: multiple string-rendering route modules (detected in audit)
   Files: various app_core/*.py and app_core/routes/*.py
   Evidence:
     - HTML audit flagged forms missing hidden csrf_token name attribute
   Root cause: Handcrafted HTML strings returned by routes omitted the CSRF hidden input required by Flask-WTF guardrails.
   Remediation: Insert <input type="hidden" name="csrf_token" value="{{ csrf_token() }}"> into every POST/PUT/DELETE form served as strings. Shortener routes updated accordingly.
   Change made: app_core/routes/shortener.py updated to include csrf_token in form strings.
   Priority: P0
   Validation: re-run html audit and confirm no POST forms flagged.

4) Symptom: Potential circular import between routes modules and home.render_layout
   Affected routes: modules importing render_layout at module top-level
   Files: app_core/routes/home.py and others
   Evidence:
     - Several modules import render_layout from app_core.routes.home at top-level; this can cause circular import if home imports other modules (observed in traces)
   Root cause: top-level imports executed at module import time; if home imports back, circularity occurs.
   Remediation: Use local import inside route functions to break circular import; apply only where circular import occurs. Shortener uses local import as fallback.
   Change made: shortener routes use local import pattern; minimal fallback render_layout defined when import fails (temporary).
   Priority: P1
   Validation: stability test and manual import checks.
